# MAGI 使用手册

> 版本: 5.0 | 更新日期: 2026-04-12

## 快速开始

### 1. 安装和启动

```bash
git clone https://github.com/harborliu-ui/magi.git
cd magi
npm install
npm run dev
```

打开浏览器访问 `http://localhost:3000`

### 2. 首次配置

进入 **设置** 页面（侧边栏底部），配置以下内容：

#### LLM 配置
- **API 地址**：填入 LLM API 地址（OpenAI 兼容或 Anthropic）
- **API Key**：填入 API Key
- **模型**：选择模型（如 gpt-4o, claude-sonnet-4-20250514 等）
- 点击"保存并测试连接"验证
- **保存失败时会显示错误信息，不会在未成功时显示「已保存」**

#### Confluence 配置
- **Base URL**：如 `https://confluence.shopee.io`
- **Token**：Bearer Token（需要**读写权限**以支持 PRD 发布功能）
- 点击"保存并测试连接"验证
- **保存失败时会显示错误，不会误报成功**

#### PRD 模板配置（v3.0 新增）
- **Confluence 模板页面 ID**：仅可输入**数字**页面 ID
- 该页面可以是一个空白模板结构或已有的 PRD 样例
- 生成 PRD 时会以该页面内容作为结构参考
- 留空则使用系统内置默认模板
- 可使用 **「测试拉取模板」** 验证当前 ID 能否成功拉取模板内容

#### Google Workspace 配置（可选）
- 配置 OAuth 凭证路径
- 首次使用前需运行 `python3.11 ~/.cursor/skills/google-workspace/scripts/auth.py` 完成授权
- **保存失败时显示错误，不会误报成功**

## 使用流程

### Step 0: 创建系统和模块

1. 在侧边栏点击 **+ 新建系统**
2. 输入系统名称（如 ISC）和描述
3. 进入系统页面，上传设计原则和系统边界（MD 文件或直接编辑）
4. 添加知识库来源（Confluence 目录 URL、页面 ID 或搜索关键词）
5. 在系统下创建模块（如 Invoice Stock）

### Step 1: 业务需求

1. 在模块下创建项目
2. 进入项目工作台，当前位于"业务需求"标签
3. 点击"添加文档"
4. 选择文档类型：
   - **BRD / FRF**：核心需求文档，AI 将深入分析
   - **Google Doc**：粘贴 Google Doc 链接，系统自动拉取内容（作为核心文档）
   - **参考材料**：辅助上下文，AI 不会单独分析
   - **外部链接**：相关链接记录
5. 文档会自动分为"核心需求文档"和"参考材料"两组显示

### Step 2: 业务分析

1. 切换到"业务分析"标签
2. 点击 **"开始分析"** 按钮
3. AI 会输出：
   - **原文标注与评论（v4.0 升级为 Google Doc 风格）**：
     - 只要有**核心需求文档**，标注相关区域即**始终可见**
     - **文档格式还原**：如果 BRD 来自 Google Doc，系统保留原始 HTML 格式，包括标题、粗体、列表、表格、颜色等
     - **侧边栏评论**：界面分为左侧文档 + 右侧评论栏，类似 Google Doc 的 Comment 功能
     - AI 在 BRD/FRF 原文中标注不清晰或有问题的文字段落，高亮显示
     - 每条评论卡片包含：引用原文、**AI 的问题**、**建议答案**、严重度标签
     - 点击文档中的高亮文字 → 自动滚动到右侧对应的评论卡片
     - 可删除不需要的标注；保存标注时界面显示进行中状态
   - **BRD/FRF 解读**：需求的结构化总结
   - **业务流程图**：Mermaid 格式的流程图（如果涉及明确流程）
   - **待确认点**：分为三个严重度等级
     - 🔴 **方案质疑**（critical）：违反设计原则或超越系统边界的需求，**必须处理**
     - 🟡 **需关注**（warning）：有潜在风险的问题
     - 🔵 **待确认**（info）：常规确认问题
   - **业务规则**：从文档中提取的明确规则

4. **添加自定义评论（v4.0 升级）**：
   - 在文档视图中直接选中 BRD 原文中的文字
   - 底部会弹出评论输入框，输入评论内容
   - 点击"评论"创建自定义标注，新标注会出现在右侧评论栏中
   - 按 Escape 可取消添加

5. **处理待确认点**：
   - 查看 AI 的建议答案，点击"采纳"使用建议
   - 或输入自己的回答
   - 点击"确认回答"仅回答问题
   - 点击"确认并转为规则"会同时将回答转为业务规则

6. **对话反馈**：
   - 在"反馈与对话"区域输入反馈
   - 例如："BRD 解读中缺少了关于 XX 的说明"
   - AI 会据此更新解读内容和流程图

7. 可以手动添加业务规则

### Step 3: 高阶方案设计

1. 切换到"高阶方案"标签
2. 点击 **"生成高阶方案"**
3. AI 会输出三个独立 section，每个 section 包含文字描述和 **Mermaid 架构图（v3.0 新增）**：
   - **信息架构**：页面改动分析 + 页面结构图
   - **系统架构**：跨系统交互和接口变更 + 序列图
   - **数据架构**：数据实体和字段变更 + ER 图

4. **受影响系统（v3.0 新增）**：
   - AI 自动识别本需求涉及的所有业务系统/模块
   - 当前项目所在的系统标记为"当前系统"
   - 若未识别到跨系统影响，界面会提示 **「未识别跨系统影响」**
   - 如果涉及多个系统，可选择 PRD 范围：
     - **仅当前系统**：PRD 只覆盖当前系统的改造内容
     - **所有受影响系统**：PRD 覆盖所有系统的改造
   - **切换范围若接口失败，会显示错误信息**

5. **Per-section 对话（v3.0 新增）**：每个 section 有独立的"反馈与对话"区域
   - 消息加载时有 **loading**；尚无消息时有 **空状态**；加载失败会 **提示错误**
   - 针对某个 section 提供反馈，AI 只更新该部分
   - 例如：在系统架构的对话区输入"缺少和 WMS 的接口说明"

6. 每个 section 可以单独编辑文字描述；**点保存时按钮显示加载态，失败会提示错误**

7. 满意后点击 **"确认方案"**；若请求失败会 **显示错误**，不会静默成功

> ⚠️ 确认方案后，该方案将作为 PRD 生成的重要输入

### Step 4: 产品需求

1. 切换到"产品需求"标签
2. 点击 **"生成 PRD"**
3. AI 会综合所有上游信息生成完整 PRD：
   - 业务需求文档
   - 已确认的业务规则
   - 已回答的确认点
   - 已确认的高阶方案
   - 系统上下文（设计原则、边界、知识库）
   - PRD 模板（Confluence 或内置模板）
4. 支持：
   - **编辑**：直接修改 PRD 内容
   - **复制 MD**：一键复制 Markdown 文本
   - **发布到 Confluence（v3.0 新增）**：
     1. 点击"发布到 Confluence"按钮
     2. 输入页面标题（默认：项目名 + PRD）
     3. 在目录浏览器中选择父页面
     4. **在最终发布前会弹出确认步骤，展示选中的父页面与将要创建的标题**
     5. 确认后点击"发布"
     6. 发布成功后自动打开 Confluence 页面
   - **重新生成**：基于最新信息重新生成

## 知识库配置指南

### Confluence 目录（推荐）
输入格式：`https://confluence.shopee.io/display/SSCP/ISC`

系统会自动：
1. 解析 URL 获取 Space Key 和页面标题
2. 找到对应页面
3. 递归获取所有子页面（最多 4 层深度）
4. 将所有页面的 ID、标题、摘要（前 500 字）存入本地索引

**索引管理（v5.0 新增）：**
- 添加 Confluence 目录后，在知识库来源列表中会显示索引状态
- 未索引时显示黄色 "未索引" 标签，点击「构建」或刷新按钮触发索引
- 索引完成后显示绿色标签：已索引页数、有内容页数、索引时间
- 首次索引大型目录（如 ISC 有 400+ 页）约需 30-90 秒
- 如 Confluence 内容有更新，点击刷新按钮重新索引

**智能检索（v5.0 新增）：**
- 分析/HLD/PRD 时，系统自动从 BRD 内容中提取关键词
- 同时在本地索引和 Confluence CQL 中搜索最相关的文档
- 合并去重后取 top 20 篇，每页最多 8000 字符
- 无需手动选择，系统自动匹配最相关的 KB 文档

### Confluence 页面 ID
直接输入数字 ID（如 `123456`），获取单个页面内容

### 关键词搜索
输入搜索关键词，系统会搜索 Confluence 并返回相关结果

## 错误日志（v4.0 新增）

当系统遇到错误时（如 LLM 返回异常、Confluence 连接失败等），错误会自动记录到日志中。

### 查看错误日志
1. 点击侧边栏「系统管理 > 错误日志」
2. 日志页面展示：
   - **统计卡片**：总计、严重、错误、警告数量，可点击过滤
   - **日志列表**：按时间倒序，展开查看详情
3. 每条日志包含：
   - **错误详情**：完整的错误消息
   - **堆栈追踪**：帮助定位错误来源
   - **请求内容**：触发错误的请求参数
   - **上下文信息**：相关的项目 ID、操作类型等
   - **原因分析**：系统自动推断可能的原因和建议处理方式

### 管理日志
- 点击单条日志右侧的删除图标删除
- 点击顶部"清空日志"批量清除所有记录

## 常见问题

**Q: Google Doc 拉取失败怎么办？**
A: 确保已在设置中配置 Google Workspace 凭证，并运行过 `auth.py` 完成 OAuth 授权。

**Q: 分析结果不准确怎么办？**
A: 使用对话反馈功能告诉 AI 哪里有偏差，AI 会更新内容。也可以重新分析。

**Q: 系统设计原则和边界有什么作用？**
A: AI 在分析时会参考这些信息，特别是用于识别"方案质疑"——违反设计原则或超越系统边界的需求。

**Q: 内联标注和待确认点有什么区别？**
A: 内联标注是直接在 BRD 原文上的批注，指向原文的具体位置；待确认点是独立的问题列表，有些问题（如业务逻辑推理出的问题）不能对应到原文的特定段落。两者可以关联。

**Q: 为什么高阶方案里显示了"受影响系统"？**
A: 一个业务需求可能涉及多个系统的改造。AI 会自动识别，用户可以决定 PRD 是否只覆盖当前系统的改造内容。

**Q: 发布到 Confluence 失败怎么办？**
A: 确保 Confluence Token 有**写入权限**（创建页面权限），并且目标 Space 和父页面可访问。

**Q: PRD 模板从哪里来？**
A: 在设置页面用**纯数字**填写 Confluence 模板页面 ID，可用 **「测试拉取模板」** 先验证。该页面的内容会作为 PRD 的结构参考。如果未配置或获取失败，使用内置默认模板。
